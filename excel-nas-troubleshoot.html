📊 Excel 大型檔案 NAS 慢速查修 SOP

金仙 ✨ 撰寫 · 適用於 Windows 11 + NAS 環境 · 同目錄下多子目錄連結

⚠️ 核心觀念：Excel 開啟含連結的檔案時，會逐一讀取每個連結檔案。 在本機 SSD 是「毫秒級」，在 NAS 上變成「網路讀取」，每一個小延遲加起來就變大延遲。 尤其是 Wi-Fi 連線，延遲會放大 10~50 倍。

①三秒快速檢測

先做這三個測試，馬上知道問題在哪一層：

把檔案從 NAS 複製到本機桌面
看傳輸速度：≥ 80 MB/s 網路正常 < 30 MB/s 網路或 NAS 問題
在桌面打開那個 Excel
變快很多 → 問題在 NAS 存取 一樣慢 → 問題在檔案本身或 Excel
開一個全新空白 Excel
很快 → 問題在舊檔案或連結 也慢 → Excel/Windows 問題

②NAS 層檢查（最常出問題）

檢查項目怎麼查解法
SMB 版本 PowerShell: Get-SmbConnection 確認 Dialect 是 3.0+
SMB 加密 Get-SmbServerConfiguration 關閉 EncryptData，加密會拖慢傳輸
🔴 連線方式 工作管理員 → 乙太網路 必須插實體線，Wi-Fi 會慢很多
網路延遲 ping NAS_IP -t 正常 < 1ms，若 > 5ms 查交換器
NAS 硬碟負載 NAS 後台看硬碟使用率 如果 100% 表示硬碟不夠快
🔴 最常見的雷：新電腦用 Wi-Fi
新買的筆電或桌機如果是走 Wi-Fi，連 NAS 開 Excel 大型檔案會非常痛苦。 Excel 讀取連結不是一次大檔案傳輸，而是「很多次小檔案讀取」， Wi-Fi 的延遲（5~20ms）比有線（<1ms）高太多，每次小讀取都在等。 解法：插實體網路線，不用改任何設定就會有感變快。

③Windows 11 / Excel 層

調整項目說明
🔴 Defender 即時掃描 將 NAS 資料夾加入 Defender 排除清單，否則每次讀寫都被掃毒拖慢
搜尋索引 關閉 Windows Search 對 NAS 位置的索引
離線檔案 Offline Files 關閉此功能（控制台 → 同步中心）
🔴 Excel 自動更新連結 Excel → 檔案 → 選項 → 進階 → 一般 → 取消勾選「要求自動更新連結」
🔴 Excel 計算選項 設為「手動」，開檔後再按 F9 手動計算
✅ 最有感的三個調整：
1️⃣ 插實體網路線（如果正在用 Wi-Fi）
2️⃣ Defender 排除 NAS 資料夾
3️⃣ Excel 計算改手動 + 取消自動更新連結

④結構性解法（前三步都試過還慢）

💡 方案A：本機快取（Windows 離線檔案）

檔案總管 → NAS 目錄 → 右鍵 → 永遠可離線使用

✓ Excel 讀取本機快取（SSD 速度）
✓ 存檔時才寫回 NAS
✓ 最不花力氣的解法

💡 方案B：檔案重構
  • 所有連結改用 相對路徑（非絕對路徑），加速連結解析
  • 大量連結改用 Power Query 匯入，減少 Excel 跨檔案連結數量
  • 把最常連的表格搬到同一個 Excel 內的不同工作表
💡 方案C：本機 SSD 工作目錄（最簡單粗暴）

工作時把整個目錄（Excel + 所有子目錄）複製到本機 SSD 操作，
做完再整個複製回 NAS。

✓ 本機 SSD 速度完勝任何 NAS
✓ 適合每天固定作業，結束時一次性同步回去

🚀 建議優先順序

  1. 插實體網路線（如果現在用 Wi-Fi，這一步就解決了）
  2. Defender 排除 NAS 資料夾
  3. Excel 計算改手動 + 取消自動更新連結
  4. NAS 資料夾設「永遠可離線使用」
  5. 還不行 → 改用本機工作目錄方案
