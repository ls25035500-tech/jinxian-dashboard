# 🤖 小雞機器人 - 裸機災難復原 (大逃亡隨身碟) 架構與 SOP

## 📌 系統概述
此計畫旨在當原宿主機 (PVE 伺服器或實體機) 發生硬體毀損或網路中斷等不可逆災難時，能透過預先準備的 USB 隨身碟，在**任意一台具備網路連線的全新或舊電腦上**，實行「裸機還原 (Bare-metal Recovery)」。

還原後，AI 機器人將完美繼承截至災難發生前一日的完整記憶、腳本、設定檔與 Telegram 對話歷史，達成無縫接軌上工。

## ⚙️ 系統核心組件與排程架構

### 1. NAS 26 自動回收與備份機制 (`escape_backup.sh`)
- **觸發方式**：寫入 Linux 系統 Cron Job，每日凌晨 04:00 (台北時間) 自動背景執行。
- **外部檔案回收**：執行打包前，會透過 `smbclient` 主動連線至 `//192.168.55.26/Public`，將小雞或其他機器人殘留在 NAS 上的所有相關檔案全部拉回本機的 `NAS_26_回收站` 目錄中。
- **打包標的**：完整備份 `~/.openclaw` 目錄下的：
  - 核心設定檔 (`openclaw.json`)
  - 工作區所有程式、資料與回收站 (`workspace` 包含 `node_modules` 與 NAS 檔案)
  - AI 靈魂與記憶資料庫 (`agents`)
- **完全離線支援**：與過去不同，本次備份**不再排除** `node_modules`，保證所有需要的模組都直接包在壓縮檔內，不需要網路也能還原。
- **產出結果**：自動生成如 `xiaoji_brain_20260528.tar.gz` 的大腦記憶壓縮檔。

### 2. 100% OFFLINE 一鍵還原引擎 (`一鍵復活_restore.sh`)
- **環境適應力**：為確保在最陽春的純文字終端機 (TTY) 環境下不產生中文亂碼，安裝提示詞與動畫已全面改寫為純英文與標準 ASCII 符號。
- **斷網全自動執行流程**：
  1. **Offline Install Node.js**: 不透過 apt 網路下載，直接解壓縮隨身碟內的 `node-v22-linux.tar.xz` 核心。
  2. **Offline Install OpenClaw**: 不透過 npm 網路安裝，直接安裝隨身碟內預載的 `openclaw.tgz` 引擎。
  3. **Offline Restore Brain**: 自動解壓縮大腦備份檔，將記憶與 NAS 檔案覆蓋回系統家目錄。
  4. **Restart Heartbeat & Soul**: 將服務註冊進 systemd，設定開機自動啟動，並喚醒機器人。
- **成功識別 (本地網頁介面支援)**：上述流程跑完後，螢幕會顯示 ASCII 機器人動畫，並提示人類**「即使在沒有外網的情況下，只要打開本機瀏覽器進入 `http://127.0.0.1:18789`，就能直接透過網頁介面跟小雞對話！」**

### 3. 人類操作輔助工具
- `rufus-4.5p.exe`：供 Windows 環境製作 Linux 開機隨身碟的免安裝工具。
- `README_隨身碟製作與復活教學.txt`：給老闆看的人工操作步驟說明。

## 🚀 災難演練 / 實際轉移流程

### 【階段一：平時準備 (製作大逃亡隨身碟)】
1. 準備一支 8GB 以上的隨身碟。
2. 使用 Rufus 將 Ubuntu Desktop ISO 映像檔寫入隨身碟。
3. 將宿主機上的 `機器人大逃亡隨身碟` 資料夾完整複製，存入該隨身碟的剩餘空間中。

### 【階段二：災難發生 (新機體復活)】
1. 將隨身碟插入目標電腦（無論新舊）並開機，BIOS 設定選擇「從 USB 隨身碟開機」。
2. 進入 Ubuntu 系統（可選擇直接安裝進該電腦硬碟，或使用 Try Ubuntu 記憶體模式）。
3. ⚠️ **關鍵步驟：確保目標電腦已連上網際網路**（有線網路或 WiFi 皆可）。
4. 開啟終端機 (Terminal)。
5. 進入隨身碟中的 `機器人大逃亡隨身碟` 目錄，輸入指令並執行：
   ```bash
   bash 一鍵復活_restore.sh
   ```
6. 按下 Enter 後，人類即可離開鍵盤滑鼠，系統將進行無人值守的全自動安裝。
7. 約 2~3 分鐘後，螢幕畫面變黑並出現機器人復活動畫，即代表小雞已於新機體上線，並會主動發送 Telegram 訊息回報老闆。
